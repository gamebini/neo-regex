고급 패턴 가이드

전문가가 알아야 할 고급 정규식 패턴들

미리보기/뒤돌아보기

고급

Lookahead와 Lookbehind를 사용하여 특정 조건을 만족하는 위치에서만 매칭을 수행합니다.

(?=.*[A-Z])(?=.*[0-9]) 대문자와 숫자를 모두 포함하는 패스워드
(?<=@)\w+ @ 뒤에 오는 단어만 매칭

비캡처 그룹

중급

그룹화는 하되 캡처하지 않는 패턴으로 성능을 향상시킬 수 있습니다.

(?:https?|ftp):// 프로토콜 부분을 그룹화하되 캡처하지 않음

원자 그룹

전문가

백트래킹을 방지하는 원자 그룹으로 성능 문제를 해결합니다.

(?>a+)b 백트래킹 없이 a+를 매칭

명명된 캡처 그룹

중급

그룹에 이름을 부여하여 더 읽기 쉽고 유지보수하기 좋은 패턴을 만듭니다.

(?P<year>\d{4})-(?P<month>\d{2}) 년도와 월을 명명된 그룹으로 캡처

성능 최적화 팁

정규식 성능을 극대화하는 전문가 노하우

재앙적 백트래킹 방지

(a+)+, (a*)* 같은 중첩된 수량자는 지수적 시간복잡도를 야기합니다.

해결책: 원자 그룹 (?>a+)+ 또는 소유적 수량자 a++ 사용

앵커 활용하기

^와 $를 적절히 사용하면 불필요한 매칭 시도를 줄일 수 있습니다.

예시: ^[a-z]+$는 전체 문자열이 소문자인지만 확인

문자 클래스 최적화

자주 사용되는 문자는 문자 클래스의 앞쪽에 배치하면 더 빠르게 매칭됩니다.

예시: [0-9a-zA-Z] 대신 [a-zA-Z0-9]

컴파일과 재사용

같은 패턴을 반복 사용할 때는 미리 컴파일해서 재사용하세요.

JavaScript: const regex = new RegExp(pattern); // 재사용
